# 프로젝트 요약

## 🎯 프로젝트 개요

**YouTube Intelligence Dashboard**는 YouTube 채널 성장을 최적화하기 위한 데이터 분석 대시보드입니다.

### 핵심 가치
- ❌ 단순 조회수 확인 → ✅ **액션 가능한 인사이트**
- ❌ 수동 데이터 수집 → ✅ **자동 수집 및 분석**
- ❌ 직감 기반 결정 → ✅ **데이터 기반 전략**

## 🚀 주요 기능

### 1. 액션 필요 섹션
```
🚨 긴급: 조회수 급락 (평균 대비 -70%)
⚠️ 주의: 좋아요율 저조 (평균 대비 -50%)
✅ 성공: 평균 대비 2배 이상 성과
```

### 2. 성공 패턴 분석
```
상위 20% 영상 vs 일반 영상
- 평균 조회수: 50,000 vs 10,000 (+400%)
- 평균 길이: 12분 vs 8분 (긴 영상이 더 잘됨)
- 최적 업로드: 월요일 18시
```

### 3. 트래픽 소스 분석
```
알고리즘 선택률: 14.7%
추천 영상: 15% → 개선 필요 (목표 20%+)
YouTube 검색: 60% → SEO 우수
```

### 4. 영상별 상세 분석
```
특정 영상 선택 → 트래픽 소스 + 채널 평균 비교
조회수: 100,000 (+250% vs 평균)
알고리즘 추천: 70% (알고리즘 선택!)
```

## 🏆 프로젝트 특징

### 1. 액션 가능한 인사이트
- 단순 데이터 조회가 아닌 실행 가능한 조언
- 자동 분류 (긴급/주의/성공)
- 구체적 추천 (업로드 시간, 영상 길이 등)

### 2. 알고리즘 중심
- 트래픽 소스에서 RELATED_VIDEO 비율 추적
- 알고리즘 선택 여부 자동 감지
- 성공 패턴으로 알고리즘 선택 조건 파악

### 3. 로컬 우선 (Local-first)
- 서버 불필요 (SQLite)
- 데이터 소유권 (로컬 저장)
- 빠른 프로토타이핑 (Streamlit)

### 4. 자동화
- 데이터 수집 자동화
- 인사이트 자동 생성
- 향후: 알림, 리포트 자동화

## 🎓 배운 점

1. **YouTube Analytics API의 강력함**
   - 트래픽 소스 = 성장 최적화 핵심
   - RELATED_VIDEO로 알고리즘 선택 파악

2. **Streamlit의 효율성**
   - 별도 프론트엔드 없이 빠른 개발
   - 멀티페이지 지원으로 확장 가능

3. **데이터 기반 의사결정**
   - 직감 대신 데이터
   - 평균과 비교로 액션 도출

4. **MVP → 고도화 프로세스**
   - 빠르게 MVP 완성
   - 사용자 피드백으로 개선
   - 점진적 기능 추가

## 🚀 빠른 시작

```bash
# 1. 패키지 설치
pip install -r requirements.txt

# 2. DB 초기화
python database/schema.py

# 3. OAuth 인증 (처음 1회)
cd poc_scripts
python poc_authenticate.py account1

# 4. 데이터 수집
python collectors/collect_all.py

# 5. 대시보드 실행
streamlit run app/dashboard.py
```
