# NotebookLM 품질 테스트 체크리스트

NotebookLM의 표/이미지 처리 능력을 검증하기 위한 테스트 가이드

---

## 📋 테스트 개요

### 목적
NotebookLM Video Overview가 우리 프로젝트에 적합한지 확인

### 핵심 검증 사항
- ✅ 표를 정확하게 슬라이드에 반영하는가?
- ✅ 이미지/차트 품질이 유지되는가?
- ✅ 한글 처리가 정확한가?
- ✅ 복잡한 레이아웃도 처리되는가?

### 소요 시간
약 1-2시간

---

## 📝 테스트 문서 준비

### 문서 1: 간단한 표 (기본 테스트)

```markdown
# 2024 부동산 시장 분석

## 지역별 거래량

| 지역 | 거래량 | 평균 가격 | 전년 대비 |
|------|--------|-----------|-----------|
| 서울 | 1,234건 | 10.5억원 | -15.3% |
| 경기 | 2,456건 | 5.2억원 | -8.7% |
| 부산 | 987건 | 4.1억원 | -12.1% |
| 대구 | 654건 | 3.8억원 | -9.5% |

## 주요 특징

- 전반적인 하락세
- 서울 낙폭 가장 큼
- 지방 상대적 안정
```

**파일명**: `test-1-simple-table.pdf`
**만드는 법**: Google Docs로 작성 → PDF 다운로드

### 문서 2: 복잡한 표 + 차트 (실전 테스트)

```markdown
# 2024년 주식 시장 분석 보고서

## 섹터별 수익률 (단위: %)

| 섹터 | Q1 | Q2 | Q3 | Q4 | 평균 | 전년 대비 |
|------|----|----|----|----|------|-----------|
| 기술 | 5.2 | 3.8 | -2.1 | 4.5 | 2.85 | +12.3% |
| 금융 | 2.1 | 1.5 | 0.8 | 2.3 | 1.68 | +5.7% |
| 헬스케어 | 3.4 | 4.2 | 3.8 | 5.1 | 4.13 | +18.2% |
| 에너지 | -1.2 | 0.5 | 1.2 | -0.8 | -0.08 | -3.5% |
| 소비재 | 1.8 | 2.3 | 1.9 | 2.7 | 2.18 | +8.1% |

**주**: 빨간색=손실, 파란색=이익, 굵은 글씨=최고 수익

## 월별 거래량 추이

[여기에 차트 이미지 삽입]
- 막대 그래프 또는 선 그래프
- 1월~12월 데이터
- 한글 레이블

## 주요 인사이트

1. 헬스케어 섹터 강세
2. 에너지 섹터 부진
3. 기술주 변동성 확대
```

**파일명**: `test-2-complex-table.pdf`
**차트 만들기**: Excel/Google Sheets → 차트 생성 → 이미지 복사

### 문서 3: 이미지 중심 (디자인 테스트)

```markdown
# 2024 마케팅 성과 리포트

## 채널별 ROI

[인포그래픽 이미지]
- 원형 차트 (채널별 비중)
- 아이콘 포함
- 컬러풀한 디자인

## 캠페인 성과 비교

| 캠페인 | 노출수 | 클릭률 | 전환율 | ROI |
|--------|--------|--------|--------|-----|
| A | 1,234,567 | 2.3% | 1.2% | 250% |
| B | 987,654 | 3.1% | 1.8% | 320% |
| C | 654,321 | 1.9% | 0.9% | 180% |

## 사용자 여정 맵

[플로우차트 이미지]
- 인지 → 관심 → 구매 → 충성
- 화살표와 아이콘
- 단계별 전환율 표시

## 결론

- 캠페인 B가 가장 효과적
- 모바일 채널 강화 필요
- Q1 대비 30% 성장
```

**파일명**: `test-3-image-heavy.pdf`
**이미지 준비**: Canva, Figma, PowerPoint 등으로 제작

## 🧪 테스트 실행

### Step 1: NotebookLM 접속

```
1. https://notebooklm.google/ 접속
2. Google 계정 로그인
3. "New Notebook" 클릭
```

### Step 2: 문서별 테스트

각 문서마다 반복:

```
1. 문서 업로드
   - Sources → Upload
   - PDF 파일 선택

2. Video Overview 생성
   - Studio 탭 클릭
   - "Video Overview" 선택
   - 설정:
     * Format: Explainer
     * Style: Whiteboard (또는 다른 스타일)
     * Language: 한국어

3. Generate 클릭

4. 대기 (3-5분)

5. 다운로드
   - Download 버튼 클릭
   - MP4 저장

6. 재생 및 평가 (아래 체크리스트 사용)
```

### 결과 해석

**25-30점: 우수** ✅
```
→ 프로젝트 진행 OK
→ NotebookLM 품질 충분
→ 설계 문서 작성 시작
→ 자동화 파이프라인 구축
```

**18-24점: 보통** ⚠️
```
→ 조건부 진행
→ 전처리 파이프라인 필요
  - 이미지 최적화
  - 표를 이미지로 변환
  - 문서 포맷 최적화
→ 추가 개발 비용 고려
```

**12-17점: 미흡** ⚠️⚠️
```
→ 신중한 판단 필요
→ 대안 검토:
  1. NotebookLM (오디오만) + 영상 편집
  2. 다른 AI 도구 (Pictory, Synthesia)
  3. 직접 영상 생성 파이프라인
→ 프로토타입 제작 후 재평가
```

**0-11점: 불량** ❌
```
→ 프로젝트 방향 전환 권장
→ 대안:
  1. 다른 AI 영상 생성 도구
  2. 수동 영상 제작 + 일부 자동화
  3. 오디오 + 슬라이드 별도 제작
→ NotebookLM 외 솔루션 탐색
```
