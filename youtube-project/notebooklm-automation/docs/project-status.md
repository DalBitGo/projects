## 🚨 핵심 검증 포인트

### NotebookLM 표/이미지 처리 능력 (검증 필요!)

#### 공식 설명
- ✅ "문서 속 이미지·다이어그램·인용구·숫자 등을 자동으로 추출"
- ✅ "표와 차트를 슬라이드에 자동 반영"

#### 실제 검증 필요 사항
- [ ] 표를 정확하게 인식하는가?
- [ ] 이미지 품질이 유지되는가?
- [ ] 커스텀 디자인이 그대로 나오는가?
- [ ] 복잡한 표(병합 셀, 색상)도 처리되는가?
- [ ] 한글 표/차트도 잘 인식하는가?
- [ ] 차트 이미지의 해상도는 적절한가?
- [ ] 디자인/레이아웃이 유지되는가?

#### 테스트 방법

```
1. 테스트 문서 3종 준비
   ├── 간단한 표 (2x3, 기본 스타일)
   ├── 복잡한 표 (10열, 색상, 병합 셀, 한글)
   └── 이미지 + 차트 혼합 (PNG, JPG)

2. NotebookLM 테스트
   - https://notebooklm.google/ 접속
   - 각 문서 업로드
   - Video Overview 생성
   - 결과 MP4 다운로드

3. 품질 평가
   - 표 정확도: ⭐⭐⭐⭐⭐
   - 이미지 화질: ⭐⭐⭐⭐⭐
   - 디자인 유지: ⭐⭐⭐⭐⭐
   - 한글 처리: ⭐⭐⭐⭐⭐

4. 결과 판단
   → 4-5점: 프로젝트 진행 OK
   → 2-3점: 전처리 파이프라인 추가
   → 0-1점: 대안 검토
```

#### 시나리오별 대응

**Case 1: 품질 우수 (4-5점)** ✅
```
→ 프로젝트 그대로 진행
→ 설계 문서 작성 시작
→ 프론트엔드 개발
```

**Case 2: 품질 보통 (2-3점)** ⚠️
```
→ 문서 전처리 파이프라인 추가
  - 표를 고해상도 이미지로 변환
  - 이미지 최적화 (해상도, 압축)
  - NotebookLM 최적 포맷으로 변환

→ 프로젝트 진행 (복잡도 증가)
```

**Case 3: 품질 불량 (0-1점)** ❌
```
→ 대안 검토
  1. NotebookLM (오디오만) + 별도 영상 편집 자동화
  2. 다른 AI 영상 생성 도구 고려
     - Pictory.ai
     - Synthesia
     - D-ID
  3. 직접 영상 생성 파이프라인 구축
     - FFmpeg로 슬라이드 영상 생성
     - TTS(Text-to-Speech)로 내레이션
```

## 🛠️ 필요한 기술 스택 (예상)

### 백엔드
- **Runtime**: Node.js 18+
- **Framework**: Express or Fastify
- **작업 큐**: Bull (Redis)
- **파일 스토리지**: Local or Cloud Storage
- **DB**: PostgreSQL or MongoDB
- **인증**: Passport.js (Google OAuth)

### 프론트엔드
- **Framework**: React or Next.js
- **스타일**: Tailwind CSS
- **상태 관리**: Zustand or Redux
- **실시간**: Socket.io (WebSocket)
- **파일 업로드**: react-dropzone
- **HTTP**: Axios

### 자동화
- **브라우저**: Puppeteer or Playwright
- **API**: googleapis (YouTube)
- **영상 처리**: FFmpeg (옵션)

### 인프라
- **개발**: Local
- **프로덕션**: VPS or Cloud (GCP, AWS, Vercel)
- **큐**: Redis
- **DB**: Managed DB or Self-hosted

## ⚠️ 주의사항

### NotebookLM Enterprise API
- ✅ Audio Overview API: 지원 명시
- ❓ **Video Overview API: 지원 여부 확인 필요**
- Enterprise 계약 없이는 API 사용 불가
- 무료 버전은 브라우저 자동화만 가능

### YouTube API 할당량
- 기본: 10,000 units/day
- 영상 업로드: 1,600 units
- **하루 최대 약 6개 영상**
- 할당량 증가 신청 가능 (수일-수주 소요)

### 브라우저 자동화 리스크
- Google 로그인: 수동 개입 필요할 수 있음
- UI 변경: NotebookLM/YouTube UI 변경 시 스크립트 수정
- Captcha: 자동화 감지 시 차단 가능
- 안정성: API 대비 불안정

## 💬 논의 내역

### 주요 질문 & 답변

**Q: NotebookLM은 영상을 만드나요, 오디오를 만드나요?**
A: 둘 다 가능합니다.
- Audio Overview: 팟캐스트 형식 (기존)
- Video Overview: 슬라이드 영상 + 내레이션 (신규)

**Q: 자동화가 가능한가요?**
A: 조건부 가능합니다.
- 무료 버전: 브라우저 자동화 (Puppeteer) - 70% 자동화
- API 버전: YouTube API (무료) + NotebookLM Enterprise (유료) - 100% 자동화

**Q: API 없이도 가능한가요?**
A: 네, 가능합니다.
- 완전 수동: 100% 무료, 시간 소요
- 브라우저 자동화: 무료, 70% 자동화

**Q: NotebookLM이 표/이미지를 그대로 영상에 넣어주나요?**
A: ⚠️ **검증 필요**
- 공식 설명: "자동으로 추출해 활용"
- 실제 품질: 테스트 필요 (핵심 검증 포인트)
- 품질에 따라 프로젝트 방향 결정

**Q: 웹 화면이 필요한가요?**
A: 사용성을 위해 권장합니다.
- CLI 버전: 빠른 검증용
- 웹 버전: 실제 사용 (권장)
- 기능: 업로드, 설정, 모니터링, 히스토리

**Q: 구현이 가능한가요?**
A: 기술적으로 가능하나, **검증 필요**
- NotebookLM 품질 테스트 필수
- 품질 확인 후 구현 시작 권장
