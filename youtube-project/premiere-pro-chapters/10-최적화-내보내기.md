### Chapter 10: 최적화 & 내보내기

#### 10.1 내보내기 기본 개념

**내보내기(Export)란?**
- 편집한 프로젝트를 최종 비디오 파일로 변환
- 플랫폼에 업로드 가능한 형식
- 압축 및 최적화

**Media Encoder 열기**
- File → Export → Media (Ctrl + M)
- 또는 Queue 버튼 클릭

#### 10.2 내보내기 설정 (YouTube 표준)

**Format (포맷)**
- **H.264**: 가장 범용적, YouTube/대부분 플랫폼
- **H.265 (HEVC)**: 더 작은 파일 크기, 고품질
- **QuickTime**: 프로 작업 (큰 파일)

**Preset (프리셋)**
- YouTube 1080p Full HD
- Match Source - High bitrate (소스와 동일)

**주요 설정 탭**

**1. Video (비디오)**
- **Width x Height**: 1920x1080 (FHD) 또는 3840x2160 (4K)
- **Frame Rate**: 시퀀스와 동일 (30fps, 60fps 등)
- **Field Order**: Progressive
- **Aspect**: Square Pixels (1.0)

**Bitrate Settings (비트레이트)**
- **Bitrate Encoding**: VBR, 1 pass (빠름) 또는 2 pass (고품질)
- **Target Bitrate**:
  - 1080p 30fps: 8-12 Mbps
  - 1080p 60fps: 12-16 Mbps
  - 4K 30fps: 35-45 Mbps
- **Maximum Bitrate**: Target의 1.5배

**2. Audio (오디오)**
- **Codec**: AAC
- **Sample Rate**: 48000 Hz
- **Channels**: Stereo
- **Bitrate**: 320 kbps (고음질)

#### 10.3 플랫폼별 내보내기 설정

**YouTube**
- Format: H.264
- Resolution: 1920x1080 (FHD) 또는 3840x2160 (4K)
- Frame Rate: 30fps 또는 60fps
- Bitrate: 8-12 Mbps (1080p)

**YouTube Shorts**
- Format: H.264
- Resolution: 1080x1920 (9:16)
- Frame Rate: 30fps
- Bitrate: 8-10 Mbps

**TikTok**
- Format: H.264
- Resolution: 1080x1920
- Frame Rate: 30fps
- Bitrate: 6-8 Mbps
- 최대 길이: 10분 (보통 15-60초)

**Instagram Reels**
- Format: H.264
- Resolution: 1080x1920
- Frame Rate: 30fps
- Bitrate: 5-8 Mbps
- 최대 길이: 90초

**Instagram Feed (일반 게시물)**
- Format: H.264
- Resolution: 1080x1080 (1:1) 또는 1080x1350 (4:5)
- Frame Rate: 30fps
- Bitrate: 5-8 Mbps

#### 10.4 내보내기 범위 설정

**전체 시퀀스 내보내기**
- Export Settings → Source Range: Entire Sequence

**부분 내보내기 (In/Out 포인트)**
1. 타임라인에서 In (`I`) / Out (`O`) 포인트 설정
2. Export Settings → Source Range: In to Out
3. 특정 구간만 내보내기

**개별 클립 내보내기**
- 프로젝트 패널에서 클립 선택 → Export

#### 10.5 빠른 내보내기

**Quick Export**
- Premiere Pro 2023 이후 기능
- 타임라인 우측 상단 "Quick Export" 버튼
- 사전 설정된 프리셋으로 즉시 내보내기
- YouTube, Social Media 등 선택

**장점**
- 클릭 한 번으로 내보내기
- 자동 최적화
- 초보자 친화적

#### 10.6 렌더링 최적화

**프록시(Proxy) 사용**
- 고해상도 원본 대신 저해상도 복사본으로 편집
- 편집 속도 향상
- 내보내기 시 자동으로 원본 사용

**프록시 생성**
1. 클립 우클릭 → Proxy → Create Proxies
2. Format: H.264, Resolution: 1/4
3. 생성 후 자동 전환

**프록시 토글**
- 프로그램 모니터 → Proxy Toggle 버튼
- 편집: Proxy, 미리보기: Full Res

**렌더 인/아웃 (Render In to Out)**
- 타임라인의 빨간색/노란색 바: 렌더링 필요
- Sequence → Render In to Out (Enter)
- 미리 렌더링하여 재생 부드럽게

#### 10.7 내보내기 큐 (Queue) 관리

**Media Encoder 사용**
- Export → Add to Media Encoder Queue
- 여러 영상 동시 내보내기 대기열
- 백그라운드에서 처리

**장점**
- Premiere 계속 사용 가능
- 여러 포맷 동시 내보내기
- 일괄 처리

#### 10.8 프로젝트 관리 및 백업

**프로젝트 정리**
- File → Project Manager
- 사용한 파일만 복사 (Collect Files)
- 프로젝트 경량화

**자동 저장 확인**
- Edit → Preferences → Auto Save
- 자동 저장 폴더 위치 확인
- 버전 복구 가능

**백업 전략**
1. **로컬 백업**: 외장 하드에 정기적 복사
2. **클라우드 백업**: Google Drive, Dropbox
3. **3-2-1 규칙**: 3개 복사본, 2개 매체, 1개 오프사이트

**미디어 함께 백업**

- 원본 영상, 오디오 파일도 함께 백업
- 프로젝트 폴더 구조 유지

#### 10.9 성능 최적화 팁

#### 10.12 자주 하는 실수와 해결법

**실수 1: 내보내기 시 화질이 떨어짐 (원본보다 흐릿함)**
- **원인**: Bitrate 설정 너무 낮음, 잘못된 코덱 선택
- **해결**:
  - 1080p 기준 최소 8 Mbps 사용
  - VBR 2-pass 인코딩 선택 (CBR보다 효율적)
  - H.264 High Profile 사용
  - Maximum Bitrate를 Target의 1.5배로 설정
- **확인**: 내보낸 영상을 100% 확대하여 디테일 체크
- **권장 설정**: Target 8 Mbps, Maximum 12 Mbps

**실수 2: 내보낸 영상의 오디오가 깨지거나 찌그러짐**
- **원인**: 오디오 레벨이 0dB 초과 (클리핑)
- **해결**:
  - Audio Track Mixer에서 Master 트랙 레벨 확인
  - 피크가 -3dB 이하로 유지되도록 조정
  - Limiter 이펙트 적용 (Hard Limiter, -1dB)
  - Essential Sound → Loudness → Auto-Match 사용
- **예방**: 편집 중 항상 오디오 미터 모니터링
- **기준**: 평균 -12dB, 최대 -3dB

**실수 3: 렌더링이 중간에 멈추거나 실패함**
- **원인**:
  - 디스크 공간 부족
  - 손상된 소스 파일
  - 무거운 이펙트 과다 사용
  - 메모리 부족
- **해결**:
  1. 저장 위치에 충분한 여유 공간 확보 (최소 영상 크기의 3배)
  2. 타임라인에서 문제 클립 찾기:
     - 렌더링 진행도 확인하여 멈춘 지점 파악
     - 해당 클립 교체 또는 재임포트
  3. 복잡한 이펙트 구간 미리 렌더링:
     - Sequence → Render In to Out
     - 렌더링된 구간은 내보내기 시 재처리 안 함
  4. Premiere 재시작 후 재시도
- **임시 방법**: 문제 구간 건너뛰고 여러 부분으로 나눠 내보내기 후 합치기

**실수 4: 플랫폼에 맞지 않는 해상도/비율로 내보냄**
- **원인**: 시퀀스 설정과 내보내기 설정 혼동
- **해결**:
  - YouTube (가로): 1920x1080 (16:9)
  - YouTube Shorts/Instagram Reels: 1080x1920 (9:16)
  - Instagram Feed: 1080x1080 (1:1)
  - TikTok: 1080x1920 (9:16)
  - Twitter: 1280x720 이상 (16:9)
- **팁**: 각 플랫폼별 프리셋 저장
  - Export Settings → Preset → Save Preset
  - 이름: "YouTube_1080p_8Mbps", "Shorts_9x16" 등
  - 다음부터 클릭 한 번으로 적용

**실수 5: 내보낸 파일이 일부 기기에서 재생 안 됨**
- **원인**: 호환성 낮은 코덱/프로필 사용
- **해결**:
  - Format: H.264 (가장 범용적, 모든 기기 지원)
  - Profile: High (대부분 기기 지원)
  - Level: 4.2 (Full HD 권장)
  - Frame Rate: 원본과 동일 (30fps 또는 24fps)
- **피해야 할 것**:
  - HEVC/H.265 (구형 기기 미지원)
  - ProRes (Mac 전용, 파일 크기 큼)
  - VP9 (웹 전용)
- **테스트**: 스마트폰, 태블릿, TV에서 재생 확인

**실수 6: Media Encoder 큐에 잘못된 시퀀스 추가**
- **원인**: 여러 시퀀스 작업 중 혼동
- **해결**:
  - 내보내기 전 시퀀스 이름 다시 확인
  - 타임라인 활성화 후 Ctrl+M (현재 시퀀스만 내보냄)
  - Media Encoder에서:
    - Source 확인 (정확한 시퀀스인지)
    - Duration 확인 (예상 길이와 일치하는지)
  - 잘못 추가 시: 큐에서 삭제 후 다시 추가
- **팁**: 시퀀스 이름을 명확하게
  - 예: "Main_Final_1080p", "Shorts_Version_9x16"

**실수 7: 내보낸 영상에 검은 테두리 (Letterbox/Pillarbox) 생김**
- **원인**: 시퀀스 비율과 소스 비율 불일치
- **해결**:
  1. **Scale to Frame Size 사용**:
     - 클립 우클릭 → Scale to Frame Size
     - 자동으로 화면에 꽉 차게 조정
  2. **수동 조정**:
     - Effect Controls → Scale: 증가 (검은 테두리 사라질 때까지)
     - Position 조정 (중요 부분 중앙 배치)
  3. **내보내기 설정 확인**:
     - Export Settings → Video 탭
     - Width/Height가 시퀀스와 일치하는지 확인
     - "Scale to Fill" 옵션 사용 (일부 버전)
- **예방**: 촬영 시 타겟 비율에 맞춰 프레이밍

#### 10.13 프로 팁

**팁 1: 프리셋 라이브러리 구축으로 내보내기 시간 90% 단축**
- **현재 방식**: 매번 설정 입력 (5-10분 소요)
- **프로 방식**: 사전 저장 프리셋 (10초 소요)
- **구축 방법**:
  1. Export Settings에서 자주 쓰는 설정 완성
  2. Preset 탭 → Save Preset
  3. 이름: "YT_1080p_8M", "Shorts_9x16_10M", "IG_1080sq" 등
  4. 프리셋 폴더 백업 (재설치 대비)
- **권장 프리셋 6종**:
  - YouTube 1080p (16:9, 8 Mbps)
  - YouTube 4K (3840x2160, 20 Mbps)
  - Shorts/Reels (1080x1920, 10 Mbps)
  - Instagram Feed (1080x1080, 8 Mbps)
  - 고품질 백업 (ProRes 422)
  - 미리보기용 저용량 (540p, 2 Mbps)

**팁 2: Media Encoder 워치 폴더로 완전 자동화**
- **개념**: 특정 폴더에 파일 넣으면 자동으로 인코딩
- **설정 방법**:
  1. Media Encoder 열기
  2. File → Add Watch Folder
  3. 폴더 선택 (예: D:/Export_Auto/)
  4. 출력 프리셋 지정
  5. 자동 인코딩 시작 간격 설정 (30초)
- **활용 시나리오**:
  - 밤사이 여러 영상 일괄 내보내기
  - 팀 작업 시 공유 폴더 사용
  - 렌더팜처럼 활용
- **주의**: 컴퓨터 전원 켜져 있어야 함

**팁 3: 2-Pass 인코딩으로 동일 용량에 화질 20% 향상**
- **1-Pass vs 2-Pass**:
  - 1-Pass: 한 번에 인코딩 (빠름)
  - 2-Pass: 첫 번째에 분석, 두 번째에 최적 인코딩 (느리지만 고품질)
- **설정**:
  - Export Settings → Video
  - Bitrate Encoding: **VBR, 2 pass**
  - Target Bitrate: 8 Mbps
  - Maximum Bitrate: 12 Mbps
- **효과**:
  - 복잡한 장면: 높은 bitrate 자동 할당
  - 단순한 장면: 낮은 bitrate로 절약
  - 전체 용량 동일, 화질 향상
- **시간**: 렌더링 약 2배 소요 (10분 영상 = 20분 → 40분)
- **권장 상황**: 최종 버전, YouTube 업로드용

**팁 4: 하드웨어 인코딩으로 렌더링 속도 5배 증가**
- **소프트웨어 vs 하드웨어 인코딩**:
  - Software: CPU 사용, 느리지만 화질 최고
  - Hardware (NVENC/QuickSync): GPU 사용, 빠르지만 화질 약간 낮음
- **활성화 방법**:
  - Export Settings → Video
  - Performance: Hardware Encoding
  - 또는 Encoding Settings → Use Hardware Encoding
  - GPU 필요: NVIDIA GTX 1050 이상 또는 Intel 6세대 이상
- **속도 비교**:
  - Software: 10분 영상 = 25분 렌더링
  - Hardware: 10분 영상 = 5분 렌더링
- **화질 차이**: 8 Mbps 이상에서는 거의 구분 불가
- **권장**: 빠른 수정본/미리보기는 Hardware, 최종본은 Software

**팁 5: LUT 적용하여 내보내기 (색보정 구워넣기)**
- **시나리오**: 외부 플랫폼용 특정 색감 필요
- **방법**:
  1. Export Settings → Effects 탭
  2. Lumetri → Enable
  3. Input LUT: 없음
  4. Creative → Look → LUT 파일 선택
  5. 내보내기 시 LUT 자동 적용
- **장점**:
  - 타임라인 수정 없이 내보내기만 변경
  - 같은 영상 여러 색감 버전 제작
- **활용**:
  - 플랫폼별 색감 (Instagram: 밝고 화사 / YouTube: 자연스러움)
  - A/B 테스트 (어떤 색감이 클릭률 높은지)

**팁 6: 인트로/아웃트로 자동 삽입**
- **문제**: 매 영상마다 인트로/아웟트로 수동 추가 번거로움
- **해결**: Premiere Pro 프리셋에 포함
- **방법**:
  1. 인트로 영상을 별도 파일로 준비 (예: intro_5sec.mp4)
  2. 시퀀스 앞뒤에 항상 배치
  3. 전체 시퀀스 내보내기
- **고급 - Media Encoder 사용**:
  1. Intro/Outro 파일 준비
  2. Media Encoder → 큐 추가
  3. Stitch 기능으로 자동 연결
     - Intro.mp4 + Main.mp4 + Outro.mp4 = Final.mp4
  4. 한 번에 렌더링
- **시간 절약**: 영상 1개당 5분 → 30개 영상 시 2.5시간 절약

**팁 7: 렌더링 중 컴퓨터 사용 가능하게 설정**
- **문제**: 렌더링 중 컴퓨터 느려서 다른 작업 불가
- **해결**:
  1. **우선순위 조정**:
     - 작업 관리자 → Media Encoder 프로세스
     - 우선순위 → 보통 이하로 변경
     - CPU 점유율 감소, 다른 작업 가능
  2. **Maximum Render Quality 끄기** (미리보기용):
     - Export Settings → Video
     - Maximum Render Quality 체크 해제
     - 렌더링 속도 30% 향상
  3. **프록시 활용**:
     - 프록시 모드로 편집 후 내보내기 시 자동 원본 사용
     - 편집 중 부담 감소
- **야간 렌더링**: Media Encoder 큐에 여러 개 추가 후 퇴근

**기법 2: LUT 적용 및 다중 색감 버전 제작**

**방법**:
1. **타임라인은 그대로, 내보내기 설정만 변경**

2. **3가지 버전 생성**

   - 같은 시퀀스를 3번 추가
   - 각각 다른 LUT 프리셋 적용
   - 파일명: video_original.mp4, video_cinematic.mp4, video_vintage.mp4

**활용**:
- A/B 테스트 (어떤 색감이 반응 좋은지)
- 클라이언트 선택지 제공
- 플랫폼별 색감 차별화

**기법 3: 인트로/아웃트로 자동 합성**

**시나리오**: 메인 영상 앞뒤에 인트로/아웃트로 자동 붙이기

**방법 1: Premiere Pro Nest 사용**
1. 별도 시퀀스 생성: "Final_with_Intro_Outro"
2. 인트로 클립 배치 (5초)
3. 메인 시퀀스를 Nest로 삽입
4. 아웃트로 클립 배치 (5초)
5. 전체 내보내기

**방법 2: Media Encoder Stitch 기능**
1. Media Encoder 큐에 추가:
   - Intro.mp4
   - Main.mp4
   - Outro.mp4
2. 3개 항목 선택 → 우클릭 → Stitch Clips
3. 자동으로 하나의 파일로 합쳐짐
4. 프리셋 적용 후 Export

**장점**:
- 타임라인 수정 불필요
- 인트로/아웃트로 변경 시 쉽게 교체
- 여러 영상에 일괄 적용 가능

**기법 4: 프록시 워크플로우 완벽 활용**

**전체 흐름**:
1. **프록시 생성**
   - 4K 원본 클립 임포트

2. **프록시로 편집**
   - Program Monitor → Proxy 버튼 활성화
   - 720p로 편집 (빠르고 부드러움)
   - 색보정, 이펙트 모두 적용 가능

3. **내보내기 시 자동 원본 사용**
   - Export Settings → Use Previews 체크 해제
   - 자동으로 4K 원본으로 교체되어 렌더링
   - 최종 영상: 4K 고화질

**장점**:
- 편집 속도 3-5배 향상
- 저사양 컴퓨터에서도 4K 편집 가능
- 내보내기는 고화질 유지

**주의**:
- 프록시 파일 저장 공간 필요 (원본의 약 30%)
- 첫 생성 시간 소요 (10분 영상 = 3-5분)

**시나리오**: 로고, 자막, 그래픽을 투명 배경으로 내보내기

**방법**:
1. **시퀀스 설정 확인**

2. **Export Settings**
   - Format: QuickTime

   - Depth: Millions of Colors+
   - 또는 PNG Sequence (여러 장의 이미지)

3. **Alpha Channel 활성화**
   - Video 탭 → Depth: Millions of Colors+ (알파 포함)
   - 렌더링 시 투명 영역 유지

**활용**:
- After Effects로 가져가기
- 다른 영상 위에 겹치기
- 라이브 방송 오버레이

**주의**: 파일 크기 매우 큼 (ProRes)

**기법 7: 오디오만 별도 내보내기 (팟캐스트, 오디오 버전)**

**시나리오**: 영상에서 오디오만 추출하여 MP3/WAV로 내보내기

**방법**:
1. **Export Settings**
   - Format: MP3 또는 WAV
   - Audio Codec: MP3 (압축) 또는 WAV (무손실)
   - Sample Rate: 48000 Hz
   - Channels: Stereo
   - Bitrate: 320 kbps (고품질 MP3)

2. **Export Audio Only**
   - Export Video 체크 해제
   - Export Audio 체크
   - 파일명: video_audio.mp3

**활용**:
- 영상을 팟캐스트로 재활용
- 오디오북 제작
- BGM 추출
- 클라이언트에게 오디오 트랙 전달

#### 10.16 문제 해결

**문제 1: "Codec Not Found" 또는 "Could not compile movie" 에러**

**증상**:
- 내보내기 시작 직후 실패
- 에러 메시지: "Codec not available" 또는 "Could not compile"

**원인**:
- 시스템에 없는 코덱 시도
- 손상된 소스 파일
- 프리셋 설정 오류

**해결책**:
1. **기본 프리셋 사용**:
   - Export Settings → Preset 드롭다운
   - "YouTube 1080p Full HD" 같은 기본 프리셋 선택
   - Custom 설정 대신 검증된 프리셋 사용

2. **코덱 재선택**:
   - Format: H.264 (가장 안전)
   - Video Codec: H.264 확인
   - 다른 코덱 시도 금지

3. **소스 파일 확인**:
   - 타임라인에서 특정 클립이 빨간색 (Missing)이면 재링크
   - 손상된 클립 교체

4. **Premiere 재시작**:
   - 프로젝트 저장
   - Premiere 종료 후 재실행
   - 캐시 삭제: Edit → Preferences → Media Cache → Clean

**문제 2: 내보낸 파일의 색감이 타임라인과 다름 (색상 시프트)**

**증상**:
- Premiere에서는 색감 정상
- 내보낸 파일은 색이 탁하거나 채도 높음
- 특히 QuickTime Player에서 심함

**원인**:
- 색 공간 불일치 (Rec. 709 vs sRGB)
- QuickTime의 색 관리 문제
- GPU 가속 설정 차이

**해결책**:
1. **Export Settings 색 공간 설정**:
   - Video 탭 → Color Space: Rec. 709
   - 웹/YouTube 표준 색 공간

2. **VLC로 재생하여 확인**:
   - QuickTime Player 대신 VLC 사용
   - VLC가 색 공간 정확하게 표시

3. **"Render at Maximum Depth" 활성화**:
   - Export Settings → Video
   - Render at Maximum Depth 체크
   - 색 정보 손실 최소화

4. **프리뷰 파일 사용 안 함**:
   - Use Previews 체크 해제
   - 원본 소스부터 다시 렌더링

**확인**:
- YouTube 업로드 후 확인 (최종 확인 방법)
- 여러 기기(PC, 스마트폰, TV)에서 테스트

**문제 3: 내보내기 속도가 너무 느림 (실시간의 10배 이상 소요)**

**증상**:
- 10분 영상 내보내기에 2시간 이상
- 진행 속도: 0.5 fps 이하
- CPU/GPU 점유율 100%

**원인**:
- 무거운 이펙트 (Warp Stabilizer, Lumetri 등)
- 4K 이상 고해상도
- 하드웨어 성능 부족
- Maximum Render Quality 활성화

**해결책**:
1. **무거운 이펙트 제거 또는 미리 렌더링**:
   - Warp Stabilizer 적용 구간: Sequence → Render In to Out
   - 녹색 렌더 바 생성 후 내보내기
   - 재처리 안 함 → 속도 향상

2. **Maximum Render Quality 끄기**:
   - Export Settings → Video
   - Maximum Render Quality 체크 해제
   - 화질 차이 거의 없음, 속도 30% 향상

3. **하드웨어 인코딩 사용**:
   - Encoding Settings → Performance: Hardware Encoding
   - NVIDIA GPU 또는 Intel QuickSync 활용
   - 속도 3-5배 향상

4. **해상도 낮추기**:
   - 4K → 1080p로 변경
   - 대부분 플랫폼에서 1080p 충분

5. **프록시 모드로 편집했는지 확인**:
   - Program Monitor → Proxy 버튼 꺼짐 확인
   - 내보내기 시 원본 사용

**임시 방법**:
- 야간 렌더링 (자기 전 시작)
- 구간 나눠 렌더링 후 결합

**문제 4: 내보낸 영상에 오디오 없음**

**증상**:
- 영상은 정상, 오디오만 없음
- 또는 특정 구간만 오디오 누락

**원인**:
- Export 설정에서 Audio 체크 해제
- 오디오 트랙 뮤트 상태
- 오디오 코덱 미지원

**해결책**:
1. **Export Settings 확인**:
   - Audio 탭 → Export Audio 체크 확인
   - Audio Codec: AAC 선택
   - Channels: Stereo

2. **타임라인 오디오 트랙 확인**:
   - 타임라인에서 M (뮤트) 버튼 해제
   - 모든 오디오 트랙 활성화

3. **오디오 클립 존재 확인**:
   - 타임라인 오디오 트랙에 클립 있는지 확인
   - Audio Meter에서 소리 파형 표시되는지 확인

4. **재내보내기**:
   - 프로젝트 저장 후 Premiere 재시작
   - 다시 Export

**문제 5: "Disk Full" 에러 (디스크 공간 부족)**

**증상**:
- 렌더링 70-90% 진행 후 중단
- 에러 메시지: "Disk is full" 또는 "Not enough space"

**원인**:
- 내보내기 위치 드라이브 여유 공간 부족
- 임시 파일 축적
- 캐시 파일 과다

**해결책**:
1. **다른 드라이브로 변경**:
   - Export Settings → Output Name
   - 저장 위치를 여유 공간 많은 드라이브로 변경
   - 예: C: → D: 또는 외장 하드

2. **불필요한 파일 삭제**:
   - 이전 렌더링 파일 삭제
   - 다운로드 폴더 정리
   - 휴지통 비우기

3. **캐시 파일 정리**:
   - Edit → Preferences → Media Cache
   - Delete Unused Cache Files
   - 수 GB 확보 가능

4. **필요 공간 계산**:
   - 내보낼 영상 예상 크기의 3배 확보
   - 10분 1080p (600MB) → 최소 2GB 여유 공간

**문제 6: 업로드 후 플랫폼에서 "Invalid File" 에러**

**증상**:
- 영상은 정상 재생
- YouTube/Instagram 업로드 시 거부됨
- 에러: "Invalid format" 또는 "Not supported"

**원인**:
- 플랫폼 제한 초과 (길이, 용량, 해상도)
- 지원하지 않는 코덱
- 메타데이터 문제

**해결책**:
1. **플랫폼별 제한 확인**:
   - **YouTube**: 최대 256GB, 12시간
   - **Instagram Reels**: 최대 90초, 4GB
   - **TikTok**: 최대 10분, 287.6MB (iOS), 72MB (Android)

2. **코덱 변경**:
   - Format: H.264 (가장 범용적)
   - Profile: Main 또는 High
   - Level: 4.2 이하

3. **용량/길이 조정**:
   - Bitrate 낮추기 (8 Mbps → 6 Mbps)
   - 영상 길이 단축
   - 해상도 낮추기 (4K → 1080p)

4. **재인코딩**:
   - Handbrake 같은 별도 툴로 재인코딩
   - "Web Optimized" 옵션 활성화

**문제 7: 렌더링 후 특정 구간만 화질 저하 (블록 현상)**

**증상**:
- 전체적으로는 화질 양호
- 빠른 움직임 또는 어두운 장면에서만 블록/모자이크

**원인**:
- Bitrate 부족 (복잡한 장면 처리 못 함)
- CBR 사용 (고정 Bitrate)
- 압축 아티팩트

**해결책**:
1. **VBR 2-pass 사용**:
   - Bitrate Encoding: VBR, 2 pass
   - 복잡한 장면에 자동으로 높은 bitrate 할당

2. **Bitrate 증가**:
   - Target Bitrate: 8 → 12 Mbps
   - Maximum Bitrate: 12 → 18 Mbps

3. **GOP Size 조정**:
   - Export Settings → Video
   - Keyframe Distance: 자동 (기본값)
   - 너무 크면 품질 저하

4. **색 공간 확인**:
   - Rec. 709 사용
   - 색 정보 압축 최소화

**예방**:
- 빠른 움직임 장면 미리 확인
- 테스트 렌더링으로 사전 점검

## 5단계: 실전 프로젝트
