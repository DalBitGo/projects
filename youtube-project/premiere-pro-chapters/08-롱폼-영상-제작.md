### Chapter 8: 롱폼 영상 제작

#### 8.1 롱폼 콘텐츠 이해

**롱폼이란?**
- 5분 이상의 긴 형식 영상
- YouTube 메인 콘텐츠
- 브이로그, 튜토리얼, 리뷰, 다큐멘터리 등

**롱폼 vs 쇼츠**
- **쇼츠**: 빠른 리듬, 짧은 메시지
- **롱폼**: 스토리텔링, 깊이 있는 내용, 느린 템포

#### 8.2 B-roll 활용법

**B-roll이란?**
- 메인 영상 (A-roll) 위에 삽입하는 보조 영상
- 시각적 다양성 제공
- 이야기를 보완하거나 강조

**B-roll 예시**
- 제품 리뷰: 제품 클로즈업 샷
- 여행 브이로그: 풍경, 거리 장면
- 요리 영상: 재료, 조리 과정

**B-roll 삽입 방법**
1. A-roll (메인 대사) 타임라인에 배치
2. B-roll을 V2 트랙에 배치
3. A-roll 오디오는 유지, 비디오만 B-roll로 교체
4. 자연스러운 타이밍에 배치

**B-roll 편집 팁**
- 2-5초 길이 권장
- 너무 자주 바뀌면 산만함
- 대사 내용과 연관된 영상 사용
- 트랜지션 없이 컷으로 전환 (자연스러움)

#### 8.3 멀티캠 편집 (Multicam)

**멀티캠이란?**
- 여러 카메라로 동시 촬영한 영상 편집
- 앵글 전환으로 역동적인 영상 제작
- 인터뷰, 라이브 공연, 팟캐스트 등

**멀티캠 시퀀스 생성**
1. 동시에 촬영한 클립들 선택 (2개 이상)
2. 우클릭 → Create Multi-Camera Source Sequence
3. Synchronize Point:
   - **Audio**: 오디오 파형으로 동기화 (가장 정확)
   - **In Point**: 클립 시작점 기준
   - **Timecode**: 타임코드 기준
4. OK → 멀티캠 시퀀스 생성

**멀티캠 편집**
1. 멀티캠 시퀀스를 타임라인에 배치
2. 프로그램 모니터에서 Multi-Camera 버튼 클릭
3. 재생하며 1, 2, 3 키로 카메라 전환
   - 1: 카메라 1
   - 2: 카메라 2
   - 3: 카메라 3
4. 실시간으로 앵글 전환 기록

**수동 조정**
- 타임라인에서 멀티캠 클립 우클릭
- Multi-Camera → 원하는 카메라 선택

#### 8.4 스토리 구성 (3막 구조)

**영상 스토리텔링 기본**

**1막: 도입 (0-15%)**
- 후크(Hook): 시청자 관심 끌기
- 주제 소개
- 무엇을 다룰지 명확히 전달

**2막: 전개 (15-85%)**
- 메인 콘텐츠
- 정보 전달, 스토리 진행
- 클라이맥스 빌드업

**3막: 결말 (85-100%)**
- 요약 및 마무리
- Call to Action (구독, 좋아요)
- 다음 영상 예고 (선택)

**예시: 10분 제품 리뷰 영상**
- 0:00-1:00 (1막): 제품 소개, 왜 이 제품인지
- 1:00-8:30 (2막): 사용 경험, 장단점, 데모
- 8:30-10:00 (3막): 총평, 추천 여부, 마무리 멘트

#### 8.5 챕터 마커 (YouTube 챕터)

**YouTube 챕터란?**
- 영상을 섹션별로 구분
- 시청자가 원하는 부분으로 건너뛰기 가능
- 시청 경험 향상

**Premiere에서 챕터 설정**
1. 중요한 구간마다 마커 추가 (`M` 키)
2. 마커 더블클릭 → 이름 입력
   - 예: "소개", "언박싱", "사용 후기", "마무리"
3. 내보내기 시 마커 포함 옵션 선택
4. YouTube 업로드 후 설명란에 타임스탬프 추가:
   ```
   0:00 소개
   1:23 언박싱
   3:45 사용 후기
   8:30 마무리
   ```

#### 8.6 롱폼 편집 워크플로우

**10분 브이로그 제작 (전체 2시간)**

**1. 소재 정리 (20분)**
- 촬영한 클립 모두 임포트
- 프로젝트 패널에서 폴더별 정리
- 사용할 클립 선별

**2. 러프 컷 (30분)**
- 시간 순서대로 클립 배치
- 대략적인 스토리 흐름 만들기
- 불필요한 부분 대략 삭제

**3. 파인 컷 (30분)**
- 정밀하게 자르기
- B-roll 삽입
- 리듬 조정

**4. 자막 & 그래픽 (15분)**
- 주요 멘트 자막
- 타이틀, 아웃트로

**5. 컬러 & 오디오 (15분)**
- 컬러 그레이딩
- BGM, 효과음
- 오디오 믹싱

**6. 최종 검토 (10분)**
- 전체 재생
- 오류 확인
- 내보내기

#### 8.7 인트로 & 아웃트로

**인트로 (Intro)**
- 5-10초 짧게
- 채널 로고, 음악
- 브랜딩 요소

**아웃트로 (Outro)**
- 15-20초
- 구독 유도
- 다른 영상 추천 (End Screen)
- "구독과 좋아요 부탁드립니다"

**Essential Graphics 템플릿 활용**
- Browse → Intro/Outro 검색
- 템플릿 드래그하여 사용
- 텍스트만 수정

#### 8.9 자주 하는 실수와 해결법

**실수 1: 롱폼 영상이 지루함 (시청자 이탈률 높음)**
- **원인**: 단조로운 편집, 페이싱 문제, B-roll 부족
- **해결**:
  - 7초마다 화면 전환 (클립 전환 또는 B-roll 삽입)
  - 음악 비트에 맞춰 편집 (리듬감 부여)
  - 불필요한 침묵/반복 구간 과감히 삭제
  - B-roll을 메인 화면의 30% 이상 사용
- **측정**: YouTube Analytics → 평균 시청 시간 확인
- **목표**: 40% 이상 시청 지속률

**실수 2: 스토리 구조 없이 시간 순서대로만 배치**
- **원인**: 기획 단계 생략, 영상 구성 의식 부족
- **해결**:
  - 3막 구조 필수 적용:
    - 1막 (도입): 흥미 유발, Hook (전체의 10%)
    - 2막 (전개): 메인 콘텐츠, 문제/해결 (전체의 70%)
    - 3막 (결말): 정리, CTA (전체의 20%)
  - 편집 전 종이에 구조 스케치
  - 각 막의 목적 명확히 정의
- **예시**: 5분 영상 = 인트로 30초 + 메인 3분 30초 + 아웃트로 1분

**실수 3: 챕터/섹션 구분 없이 하나로 쭉 이어짐**
- **원인**: 긴 영상을 하나의 덩어리로 인식
- **해결**:
  - 3-5분마다 시각적 전환 (타이틀 카드, 트랜지션)
  - YouTube 챕터 마커 추가 (설명란에 타임스탬프)
  - 섹션별 다른 BGM 사용
  - "다음은..." 같은 전환 멘트
- **효과**: 시청자가 원하는 부분 빠르게 찾기 가능

**실수 4: 오디오 레벨이 일관되지 않음 (볼륨 들쭉날쭉)**
- **원인**: 클립마다 녹음 환경 다름, 정규화 미적용
- **해결**:
  - 모든 대화 클립을 -12dB ~ -6dB로 정규화
  - Essential Sound → Auto-Match 기능 사용
  - BGM은 대화보다 20dB 낮게 (-30dB ~ -25dB)
  - 오디오 덕킹 자동화 (사이드체인 압축)
  - 마스터 트랙에 Limiter 적용 (최대 -1dB)
- **검증**: 전체 재생하며 볼륨 조절 없이 들어보기

**실수 5: 색보정 없이 원본 그대로 사용**
- **원인**: 시간 부족, 색보정 필요성 모름
- **해결**:
  - 최소한 Lumetri Color의 Auto 기능 사용
  - 모든 클립에 조정 레이어로 일괄 색보정 적용
  - 기본 설정:
    - Contrast: +10 ~ +20
    - Saturation: +10 ~ +15
    - Shadows: +5 ~ +10 (어두운 부분 밝게)
  - 30초 투자로 퀄리티 30% 향상
- **프리셋**: 한 번 만든 Look을 저장하여 재사용

**실수 6: B-roll이 메인 내용과 관련 없음**
- **원인**: 무작위로 배치, B-roll의 목적 이해 부족
- **해결**:
  - B-roll은 대사를 "보여주기" 위한 것
  - 예: "커피를 마셨다" 말할 때 → 커피 마시는 B-roll
  - 3초 이상 B-roll 지속 시 지루함 (1-2초가 적당)
  - B-roll 없는 긴 토크 구간 = 시청자 이탈
- **팁**: B-roll 촬영 시 최소 5초씩 녹화 (편집 여유)

**실수 7: 파일 용량이 너무 커서 업로드/공유 어려움**
- **원인**: 고용량 코덱 사용, 불필요한 고화질
- **해결**:
  - 내보내기 설정:
    - Bitrate Encoding: VBR 2-pass (1-pass보다 효율적)
    - Target Bitrate: 8 Mbps (1080p 기준)
    - Maximum Bitrate: 12 Mbps
  - 해상도는 1080p로 충분 (4K는 특별한 경우만)
  - 10분 영상 = 약 600MB (YouTube 업로드 쾌적)
- **비교**: CBR 20 Mbps = 1.5GB vs VBR 8 Mbps = 600MB (화질 유사)

#### 8.10 프로 팁

**팁 1: 뮤직 비트에 맞춘 컷 편집**
- **개념**: 음악의 비트나 멜로디 변화에 맞춰 화면 전환
- **방법**:
  1. BGM을 먼저 타임라인에 배치
  2. 오디오 파형 확대 (타임라인 높이 증가)
  3. 비트 위치에 마커 추가 (M 키)
  4. 마커 위치에 클립 컷/전환 배치
  5. 리듬감 있는 편집 완성
- **효과**: 시청 몰입도 25% 향상
- **주의**: 너무 많이 쓰면 정신없음 (강조 부분에만)

**팁 2: J-Cut과 L-Cut으로 자연스러운 전환**
- **J-Cut**: 다음 클립의 오디오가 먼저 나오고 영상은 나중
  - 사용: 다음 장면 예고, 대화 흐름 자연스럽게
  - 방법: V1 클립은 3초에서 끝, A1 오디오는 2.5초부터 시작
- **L-Cut**: 이전 클립의 오디오가 계속되고 영상만 전환
  - 사용: 대화 중 리액션 보여주기
  - 방법: V1 클립은 3초에서 끝, A1 오디오는 3.5초까지 연장
- **조합**: 두 기법 섞어 쓰면 전문가 느낌
- **실습**: Alt + 클릭으로 비디오/오디오 따로 이동

**팁 3: Nested Sequence로 복잡한 편집 단순화**
- **시나리오**: 10개 클립으로 만든 복잡한 인트로를 여러 프로젝트에서 재사용
- **방법**:
  1. 인트로 클립들을 모두 선택
  2. 우클릭 → Nest (중첩)
  3. 이름: "Intro_Nested"
  4. 하나의 클립처럼 취급됨
  5. 다른 프로젝트에 복사 가능
- **장점**:
  - 타임라인 정리 깔끔
  - 일괄 이펙트 적용 가능
  - 마스터 버전 수정 시 모든 인스턴스 자동 업데이트
- **단점**: 내부 수정 시 더블클릭 필요

**팁 4: 프록시 워크플로우로 편집 속도 3배 향상**
- **문제**: 4K 영상 여러 개 → 컴퓨터 느림, 미리보기 끊김
- **해결**: 저해상도 프록시 파일 자동 생성
- **설정**:
  1. 클립 우클릭 → Proxy → Create Proxies
  2. Format: H.264, Preset: 720p
  3. 자동 생성 대기 (백그라운드)
  4. Program Monitor → Proxy 버튼 활성화
  5. 편집 중엔 720p로 작업, 내보내기 시 원본 사용
- **효과**: 렉 없이 부드럽게 편집 가능
- **주의**: 프록시 생성 시간 필요 (10분 영상 = 3-5분)

**팁 5: 키프레임으로 동적인 B-roll 만들기**
- **정적 B-roll 문제**: 그냥 놓으면 지루함
- **해결**: Scale과 Position 키프레임 활용
- **예시 1 - 느린 줌인**:
  1. B-roll 클립 선택
  2. 시작점: Scale 100%
  3. 끝점: Scale 110% (2초 간격)
  4. Ken Burns 효과 (다큐멘터리 느낌)
- **예시 2 - 팬 이동**:
  1. 시작: Position X = 960
  2. 끝: Position X = 1060 (좌→우 이동)
  3. 자연스러운 흐름
- **설정**: 모든 키프레임에 Ease In/Out 적용
- **시간**: B-roll당 30초 투자로 퀄리티 2배

**팁 6: 색보정 Look 통일로 영화 느낌**
- **목표**: 모든 클립이 같은 "세계관"에 존재하는 느낌
- **방법**:
  1. 조정 레이어를 타임라인 최상단에 배치
  2. Lumetri Color 적용
  3. Creative → Look 선택 (예: Teal and Orange)
  4. 강도 조절 (50-70% 권장)
  5. 모든 클립에 일괄 적용
- **고급**: LUT 파일 사용
  - Lumetri Color → Creative → Look → Browse
  - .cube 파일 선택 (무료 LUT: free cinematic LUT 검색)
- **일관성**: 한 영상에서는 하나의 Look만 사용

**팁 7: 효율적인 타임라인 정리 - 트랙별 역할 분담**
- **프로 편집자의 타임라인 구조**:
  ```
  V5: 최종 자막/그래픽 (항상 보임)
  ```

**기법 4: 키프레임 속도 조절 (Ease In/Out 완벽 컨트롤)**

**기본 개념**: 키프레임 사이 변화 속도를 커브로 조절

**설정 방법**:
1. 키프레임 2개 생성 (예: Position, 0초에 X=0, 2초에 X=1000)
2. 키프레임 우클릭 → Temporal Interpolation
3. 옵션:
   - **Linear**: 일정한 속도 (기계적)
   - **Ease In**: 천천히 시작, 빠르게 도착
   - **Ease Out**: 빠르게 시작, 천천히 도착
   - **Ease In-Out**: 부드럽게 시작 및 종료 (가장 자연스러움)

**고급 - Bezier 커브 조절**:
1. Effect Controls 패널에서 키프레임 그래프 표시
2. Bezier 핸들 드래그하여 커브 모양 변경
3. 급격한 가속/감속 표현 가능

**실전 예시**:
- 텍스트 슬라이드 인: Ease Out (빠르게 들어와서 부드럽게 정지)
- 이미지 줌 아웃: Ease In-Out (부드러운 호흡)

**기법 5: Track Matte Key로 창의적 효과**

**개념**: 한 레이어의 밝기/알파 정보를 다른 레이어의 마스크로 사용

**사용 예시 - 텍스트로 영상 채우기**:
1. V2: 텍스트 클립 (큰 폰트, "VLOG")
2. V1: 풍경 영상
3. V2 텍스트 클립에 이펙트: Video Effects → Keying → Track Matte Key
4. Matte: V1 (풍경 영상)
5. Composite Using: Matte Luma
6. 결과: 텍스트 모양으로 풍경 영상 보임 (텍스트 안에 영상)

**응용**:
- 로고 안에 영상 재생
- 원형 마스크로 복고 느낌
- 애니메이션 마스크로 전환 효과

**기법 6: 시간 리매핑으로 속도 곡선 만들기**

**기본 Speed/Duration과 차이**: 한 클립 내에서 속도를 자유롭게 변화

**사용 방법**:
1. 클립 우클릭 → Show Clip Keyframes → Time Remapping → Speed
2. Ctrl + 클릭으로 속도 변경 지점 추가
3. 키프레임 드래그:
   - 위로: 빨라짐 (200%, 300%)
   - 아래로: 느려짐 (50%, 25%)
4. Ease In/Out 적용 (부드러운 속도 전환)

**실전 예시**:
- 점프 장면: 도약 순간 25% 슬로우 → 착지 후 정상 속도
- 액션: 평소 100% → 임팩트 순간 10% → 다시 100%
- 타임랩스 응용: 구간마다 다른 속도

**효과**: 영화적 슬로우 모션, 다이나믹한 리듬

**기법 7: Morph Cut으로 점프 컷 부드럽게**

**문제**: 긴 인터뷰에서 중간 부분 삭제 시 화면 점프 (Jump Cut)

**해결**:
1. 두 클립 사이에 컷이 있는 상태
2. Effects → Video Transitions → Morph Cut
3. 두 클립 경계에 드래그
4. AI가 자동으로 얼굴 변화를 부드럽게 보간
5. 자연스러운 전환

**주의사항**:
- 같은 장면, 비슷한 앵글에서만 효과적
- 카메라 각도 많이 바뀌면 어색함
- 렌더링 시간 오래 걸림

**대안**: 컷 위에 B-roll 덮기 (더 자연스러움)

#### 8.13 문제 해결

**문제 1: 롱폼 영상 편집 시 타임라인이 너무 복잡해서 작업 어려움**

**증상**:
- 100개 이상 클립이 타임라인에 뒤섞임
- 어떤 클립이 메인이고 B-roll인지 구분 안 됨
- 자막/그래픽 레이어 찾기 어려움

**원인**:
- 트랙 정리 안 함
- 라벨/이름 사용 안 함
- 중첩(Nest) 활용 안 함

**해결책**:

1. **트랙 역할 명확히 분담**:
   ```
   V6: 로고/워터마크
   V5: 최상위 자막
   V4: 보조 자막/그래픽
   V3: 조정 레이어
   V2: B-roll
   V1: 메인 영상
   ```
   - 우클릭 → Rename으로 트랙 이름 변경

2. **색상 라벨 사용**:
   - 메인 클립: 파란색
   - B-roll: 초록색
   - 자막: 보라색
   - 음악: 노란색
   - 클립 우클릭 → Label → 색상 선택

3. **복잡한 섹션 Nest 처리**:
   - 인트로 10개 클립 → "Intro" 네스트로 압축
   - 아웃트로 → "Outro" 네스트
   - 각 챕터별 네스트 생성

4. **Marker로 구간 표시**:
   - M 키로 마커 추가
   - 마커 이름: "1. 아침 루틴", "2. 카페", "3. 저녁"
   - 빠른 이동 가능

**예방**:
- 편집 시작 전 트랙 구조 미리 설계
- 클립 배치 시 바로 라벨 지정
- 50개 이상 클립 넘어가면 Nest 고려

**문제 2: 영상과 오디오 싱크가 점점 어긋남 (후반부로 갈수록 심해짐)**

**증상**:
- 처음에는 입 모양과 소리 일치
- 5분 이후부터 점점 밀림
- 10분 시점엔 0.5초 이상 차이

**원인**:
- 영상과 오디오 프레임레이트 불일치
- 가변 프레임레이트(VFR) 영상 사용 (스마트폰 촬영)
- 편집 중 실수로 오디오만 이동

**해결책**:

1. **VFR → CFR 변환** (가장 근본적):
   - Handbrake 또는 FFmpeg로 영상 재인코딩
   - 설정: Constant Frame Rate (CFR) 30fps
   - 변환 후 다시 임포트

2. **Premiere 내에서 재해석**:
   - 클립 우클릭 → Modify → Interpret Footage
   - Frame Rate: Assume this frame rate is 30
   - OK → 다시 타임라인 배치

3. **오디오/비디오 링크 확인**:
   - 클립 선택 → 링크 아이콘 확인 (연결 상태)
   - 끊어졌으면: 비디오와 오디오 동시 선택 → 우클릭 → Link

4. **수동 동기화**:
   - 오디오 파형 확대
   - 특정 포인트(박수, 말 시작)를 기준으로 정렬
   - 필요시 오디오 클립 미세 이동 (Alt + 좌우 화살표)

**예방**:
- 촬영 시 카메라 설정에서 고정 프레임레이트 사용
- 임포트 전 MediaInfo로 프레임레이트 확인
- 편집 중 비디오/오디오 링크 유지

**문제 3: 컴퓨터가 느려서 롱폼 편집 시 계속 렉 걸림 (미리보기 끊김)**

**증상**:
- 재생 시 프레임 드롭
- 효과 적용 시 멈춤
- 렌더 바가 노란색/빨간색

**원인**:
- 4K 영상 또는 고용량 코덱 (HEVC 등)
- RAM 부족
- 그래픽 카드 성능 부족
- 이펙트 과다 사용

**해결책**:

1. **프록시 파일 생성** (가장 효과적):
   - 클립 선택 → 우클릭 → Proxy → Create Proxies
   - Format: H.264, Resolution: 720p
   - 생성 대기
   - Program Monitor → Proxy 버튼 활성화
   - 편집은 720p로, 내보내기는 원본으로 자동 전환

2. **재생 해상도 낮추기**:
   - Program Monitor 하단 → 1/2 또는 1/4 선택
   - 화질 낮지만 편집 속도 빨라짐
   - 내보내기엔 영향 없음

3. **렌더 인/아웃 사용**:
   - 작업 영역 설정 (I/O 키)
   - Sequence → Render In to Out
   - 녹색 렌더 바 → 부드러운 재생

4. **시퀀스 설정 최적화**:
   - Sequence Settings → Video Previews
   - Codec: H.264 (가벼움)
   - Resolution: 1/2 또는 Match Source

5. **불필요한 이펙트 비활성화**:
   - 작업 중엔 색보정/블러 끄기
   - Effect Controls → fx 아이콘 클릭 (일시 비활성화)
   - 최종 단계에서만 활성화

**하드웨어 업그레이드**:
- RAM: 최소 16GB, 권장 32GB
- GPU: NVIDIA GTX 1660 이상
- SSD: 프로젝트 파일 SSD에 저장

**문제 4: 내보낸 영상 파일 크기가 너무 큼 (10분 영상이 5GB)**

**증상**:
- 10분 1080p 영상이 3-5GB
- 업로드 시간 오래 걸림
- 저장 공간 부족

**원인**:
- Bitrate 설정 너무 높음 (CBR 50 Mbps 등)
- 코덱 선택 잘못 (무손실 코덱)
- 불필요한 고화질 설정

**해결책**:

1. **적정 Bitrate 사용**:
   - Export Settings → Video
   - Bitrate Encoding: **VBR, 2 pass**
   - Target Bitrate: 8 Mbps (1080p 기준)
   - Maximum Bitrate: 12 Mbps
   - 결과: 10분 영상 = 약 600MB

2. **H.264 코덱 사용**:
   - Format: H.264 (가장 범용적)
   - Preset: YouTube 1080p Full HD
   - 화질 손실 거의 없음

3. **해상도 확인**:
   - 4K로 촬영했어도 YouTube는 1080p로 충분
   - 1920x1080으로 내보내기
   - 파일 크기 75% 감소

4. **오디오 Bitrate 조정**:
   - Audio Codec: AAC
   - Bitrate: 320 kbps (고품질)
   - 또는 192 kbps (충분)

**비교**:
- 잘못된 설정: CBR 50 Mbps, ProRes → 10분 = 4GB
- 올바른 설정: VBR 8 Mbps, H.264 → 10분 = 600MB
- 화질 차이: 눈으로 구분 어려움

**문제 5: B-roll 삽입 후 메인 영상의 오디오가 끊김**

**증상**:
- V2에 B-roll 배치
- 재생하면 메인 오디오 사라짐
- 또는 B-roll의 불필요한 오디오 재생됨

**원인**:
- B-roll 클립에 오디오가 포함됨
- 오디오 트랙 구조 이해 부족

**해결책**:

1. **B-roll 오디오 분리 및 삭제**:
   - B-roll 클립 선택 (V2)
   - Alt + 클릭으로 오디오 부분만 선택
   - Delete 키로 삭제
   - 비디오만 남음

2. **임포트 시 비디오만 가져오기**:
   - 드래그 전 Alt 누른 상태로 타임라인에 드롭
   - "Video Only" 선택
   - 오디오 없이 임포트

3. **오디오 트랙 뮤트**:
   - A2 트랙 (B-roll 오디오) 뮤트 버튼 클릭
   - 재생 시 소리 안 남

**올바른 구조**:
```
V2: B-roll (비디오만)
V1: 메인 영상 (비디오)
A1: 메인 오디오
A2: (비어 있거나 뮤트)
A3: BGM
```

**문제 6: 자막이 너무 많아서 관리가 안 됨 (100개 이상)**

**증상**:
- 10분 영상에 자막 100개
- 수정 시 하나하나 찾기 어려움
- 스타일 통일 힘듦

**원인**:
- 자막을 개별 클립으로 관리
- 템플릿 미사용
- 자막 전용 도구 활용 안 함

**해결책**:

1. **자막 파일 사용 (.srt)**:
   - 외부 프로그램(Subtitle Edit)에서 .srt 파일 작성
   - Premiere에서 Import → .srt 파일
   - 타임라인에 드래그 → 자동으로 모든 자막 배치
   - 수정: .srt 파일 편집 후 재임포트

2. **Essential Graphics 템플릿 활용**:
   - 첫 자막 완성 후 .mogrt로 저장
   - Essential Graphics → My Templates
   - 이후 드래그만으로 재사용
   - 스타일 일관성 유지

3. **자막 레이어 분리**:
   - V4: 메인 자막
   - V5: 강조 자막
   - 트랙 잠금으로 실수 방지

4. **Find 기능으로 일괄 변경**:
   - Edit → Find
   - Text 탭에서 검색어 입력
   - Replace All로 한 번에 수정

**프로 팁**:
- 자막 많은 영상은 After Effects + Dynamic Link 고려
- 또는 자막 전문 툴 (Vrew, Subtitle Edit) 사용 후 임포트

**문제 7: 내보내기 후 YouTube 업로드 시 화질이 흐릿함**

**증상**:
- Premiere에서는 선명
- YouTube 업로드 후 화질 저하
- 특히 빠른 움직임 부분 뭉개짐

**원인**:
- YouTube의 추가 압축
- 업로드 직후 저화질 처리 중
- Bitrate 너무 낮음

**해결책**:

1. **충분한 Bitrate 사용**:
   - 1080p: 8-12 Mbps (최소 8)
   - YouTube 권장: 8 Mbps
   - 고품질 원하면: 12-15 Mbps

2. **업로드 후 대기**:
   - YouTube는 업로드 직후 저화질 버전만 제공
   - HD 처리 시간: 10분 영상 = 30분~2시간
   - "HD 처리 중" 메시지 확인
   - 완료 후 다시 확인

3. **H.264 High Profile 사용**:
   - Export Settings → Video
   - Profile: High
   - Level: 4.2 또는 5.0
   - YouTube 최적화

4. **2-Pass Encoding**:
   - Bitrate Encoding: VBR, 2 pass
   - 화질 향상 (렌더링 시간 2배)

5. **색 공간 설정**:
   - Export Settings → Video
   - Color Space: Rec. 709 (YouTube 표준)

**확인 방법**:
- 업로드 후 설정 톱니바퀴 → 1080p 선택
- 통계 정보(우클릭) → "Current / Optimal Res" 확인

#### 8.14 연습 과제

**과제 1: 3막 구조 브이로그 제작 ⭐**

**목표**: 스토리 구조가 명확한 5분 브이로그 만들기

**요구사항**:
- 길이: 5분 ± 10초
- 3막 구조 필수:
  - 1막: 흥미로운 Hook (30초)
  - 2막: 메인 스토리 (3분 30초)
  - 3막: 정리 및 CTA (1분)
- B-roll 최소 5곳 삽입
- 자막 10개 이상
- BGM 및 오디오 덕킹

**평가 기준**:
- 스토리 흐름 자연스러움 (40점)
- 편집 기술 (30점)
- 오디오 믹싱 (20점)
- 시각적 완성도 (10점)

**소요 시간**: 3-4시간

**과제 2: Multicam 인터뷰 편집 ⭐⭐**

**목표**: 2개 앵글 인터뷰를 Multicam으로 편집

**요구사항**:
- 2개 카메라 영상 (정면 + 측면)
- Multicam Sequence 생성 (오디오 동기화)
- 실시간 앵글 전환 (최소 10회)
- 중요 멘트에 자막
- 색보정 일괄 적용 (조정 레이어)
- 길이: 3분

**추가 도전**:
- J-Cut/L-Cut 기법 사용
- B-roll 삽입 (인터뷰 내용 시각화)

**평가 기준**:
- Multicam 활용 숙련도 (50점)
- 편집 리듬감 (30점)
- 전환 타이밍 (20점)

**소요 시간**: 2-3시간

**과제 3: 10분 튜토리얼 영상 제작 ⭐⭐⭐**

**목표**: 무언가를 가르치는 10분 튜토리얼 영상

**요구사항**:
- 주제: 요리, DIY, 소프트웨어 사용법 등
- 길이: 10분 ± 30초
- 챕터 구분 (3-4개 섹션)
- 각 섹션마다 타이틀 카드
- 자막 30개 이상 (설명 텍스트)
- 화면 분할 또는 PIP 사용 (1곳 이상)
- 최종 체크리스트 화면
- 엔딩 CTA (구독 요청)

**고급 요구사항**:
- 모션 그래픽 인트로 (템플릿 사용 가능)
- 키프레임 애니메이션 (화살표, 강조 효과)
- 색보정 일관성
- 오디오 레벨 완벽 (-12dB ~ -6dB)

**평가 기준**:
- 전달력 및 명확성 (40점)
- 편집 완성도 (30점)
- 시각 자료 활용 (20점)
- 기술적 숙련도 (10점)
