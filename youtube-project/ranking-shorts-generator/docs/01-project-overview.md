# 랭킹 쇼츠 자동 생성기 - 프로젝트 개요 및 요구사항

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
유튜브 쇼츠의 인기 트렌드인 "랭킹 영상"을 반자동으로 생성하는 시스템 구축
- 예시: "Top 10 Football Skills", "Best TikTok Moments Ranking" 등
- 콘텐츠 수집부터 영상 편집까지 자동화하되, 사용자가 최종 콘텐츠 선택 및 검수

### 1.2 프로젝트 배경
- 유튜브 쇼츠에서 랭킹 형식의 콘텐츠 인기 증가
- 수작업으로 제작 시 많은 시간 소요 (콘텐츠 찾기, 다운로드, 편집)
- 자동화를 통한 제작 시간 단축 및 생산성 향상

### 1.3 프로젝트 목표
- TikTok에서 특정 해시태그/키워드 기반 인기 영상 자동 수집
- 웹 기반 UI를 통한 직관적인 영상 선택 인터페이스 제공
- 선택된 영상들을 자동으로 편집하여 랭킹 쇼츠 생성
- 생성된 영상에 대한 검수 프로세스 포함

---

## 2. 핵심 기능 요구사항

### 2.1 콘텐츠 수집 기능
- **입력**: 사용자가 해시태그 또는 키워드 입력
- **처리**: TikTok에서 관련 인기 영상 20~30개 검색 및 메타데이터 수집
- **출력**: 영상 목록 (썸네일, 제목, 조회수, 좋아요 수, URL)

**메타데이터 수집 항목**:
- 영상 URL
- 썸네일 이미지
- 영상 제목/설명
- 조회수
- 좋아요 수
- 업로드 날짜
- 영상 길이

### 2.2 영상 선택 및 큐레이션 기능
- **웹 UI 제공**: 수집된 영상들을 카드 형태로 표시
- **미리보기**: 각 영상의 썸네일 및 주요 정보 확인
- **선택 기능**: 5~7개 영상 선택 (체크박스 또는 클릭)
- **순서 조정**: 드래그 앤 드롭으로 랭킹 순서 변경
- **저장**: 선택된 영상 목록 및 순서 저장

### 2.3 영상 다운로드 기능
- 선택된 영상들을 로컬에 다운로드
- 고화질 옵션 선택 (720p, 1080p)
- 다운로드 진행 상황 표시 (프로그레스 바)
- 다운로드 실패 시 재시도 로직

### 2.4 영상 편집 기능
**기본 편집 작업**:
- 영상 크롭 및 리사이즈 (9:16 세로 포맷)
- 각 영상 앞에 랭킹 번호 텍스트 오버레이
  - 예: "🥇 #1", "🥈 #2", "🥉 #3", "#4", "#5"...
- 배경음악 추가 (사용자가 선택한 음원)
- 영상 간 부드러운 전환 효과
- 각 영상 길이 조정 (5~10초)

**고급 편집 옵션**:
- 커스텀 폰트 및 색상 선택
- 랭킹 텍스트 위치 조정
- 인트로/아웃트로 추가 옵션
- 자막 자동 생성 (선택 사항)

### 2.5 검수 및 출력 기능
- 생성된 영상을 `output/pending/` 폴더에 저장
- 웹 UI에서 생성된 영상 미리보기
- 승인/재작업/삭제 선택
- 승인된 영상은 `output/approved/` 폴더로 이동
- 최종 영상 다운로드 기능

## 3. 비기능 요구사항

### 3.1 성능 요구사항
- 20~30개 영상 메타데이터 수집: 30초 이내
- 5~7개 영상 다운로드: 2~5분 이내
- 최종 영상 편집 및 생성: 3~5분 이내
- 웹 UI 응답 시간: 1초 이내

### 3.2 안정성 요구사항
- TikTok API/스크래핑 실패 시 재시도 메커니즘
- 다운로드 중 네트워크 오류 처리
- 영상 편집 중 오류 발생 시 로그 기록 및 사용자 알림
- 작업 중간 상태 저장 (재시작 시 이어서 작업 가능)

### 3.3 사용성 요구사항
- 직관적인 웹 UI (기술 지식 없이도 사용 가능)
- 각 단계별 명확한 가이드 및 피드백
- 에러 발생 시 이해하기 쉬운 메시지
- 다국어 지원 (한국어, 영어)

### 3.4 확장성 요구사항
- 추후 Instagram Reels, YouTube Shorts 지원 가능하도록 설계
- 다양한 영상 템플릿 추가 가능
- 플러그인 방식으로 새로운 편집 효과 추가 가능

## 4. 시스템 제약사항

### 4.1 기술적 제약사항
- **TikTok API 제한**: 공식 API 사용 어려움 → 비공식 라이브러리(TikTokApi) 사용
- **Rate Limiting**: 과도한 요청 시 IP 차단 위험 → 요청 간격 조절 필요
- **FFmpeg 의존성**: 영상 편집을 위해 FFmpeg 필수 설치
- **저장 공간**: 다운로드 및 생성된 영상으로 인한 디스크 사용량 관리 필요

### 4.2 법적/윤리적 제약사항
- **저작권**: 다운로드한 영상의 2차 사용 시 저작권 문제 발생 가능
  - 해결책: 사용자에게 저작권 책임 고지, CCL/퍼블릭 도메인 영상 권장
- **출처 표기**: 원본 영상 크리에이터 표기 기능 추가 고려
- **TikTok 이용약관**: 스크래핑이 이용약관 위반 가능성 존재

### 4.3 운영 제약사항
- **로컬 환경 실행**: 초기에는 클라우드 배포 없이 로컬에서만 실행
- **사용자 수**: 단일 사용자(개발자 본인)만 사용 가정
- **유지보수**: TikTok 구조 변경 시 스크래퍼 코드 수정 필요

## 5. 사용자 시나리오

### 5.1 기본 사용 시나리오
1. 사용자가 웹 브라우저에서 애플리케이션 접속
2. 검색 페이지에서 해시태그 입력 (예: "football skills")
3. 시스템이 TikTok에서 관련 영상 20~30개 검색
4. 영상 목록이 카드 형태로 표시됨
5. 사용자가 마음에 드는 영상 5~7개 선택
6. 드래그 앤 드롭으로 랭킹 순서 조정
7. "영상 생성" 버튼 클릭
8. 시스템이 자동으로 다운로드 → 편집 → 생성
9. 완료되면 미리보기 페이지로 이동
10. 생성된 영상 확인 후 승인/재작업 선택
11. 승인 시 최종 파일 다운로드

### 5.2 고급 사용 시나리오
- **템플릿 변경**: 다양한 랭킹 스타일(스포츠, 코미디, 동물 등) 선택
- **음악 변경**: 기본 배경음악 대신 사용자 음원 업로드
- **텍스트 커스터마이징**: 폰트, 색상, 애니메이션 효과 변경
- **배치 작업**: 여러 주제의 영상을 동시에 생성 (큐 시스템)
